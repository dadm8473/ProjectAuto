# BM v0

BM 원칙: 패배를 팔지 않는다. 전투 중 과금 유도는 하지 않는다. 프로토타입에서는 결제를 구현하지 않고 구조만 설계한다.

## 1. Monetization Boundaries

Allowed:

- 보드 스킨
- Relay 외형
- 합성 이펙트
- 프로필 배너
- 시즌 패스 보상
- 광고 선택 보상, 전투 전/후만

Not allowed in v0:

- 전투 중 부활 판매
- 보스 실패 직전 강화 판매
- 유료 전용 강력 Relay
- 파트너보다 과금자가 더 강해지는 구조
- 확률형 유료 Supply

## 2. Currencies

|---|---|---|---|

## 6. Optional Ads

Ads are not in prototype unless explicitly required.

If added later:

- only after run
- optional double mission reward
- daily cap 3
- no combat revive
- no power reroll in active run

## 7. Player Trust Copy

Store and pass should say:

- “Cosmetic only.”
- “No paid Relay power.”
- “All gameplay systems are earned during a run.”

Avoid:

- “Increase your odds now.”
- “Beat the boss with this pack.”
- “Limited power unit.”

## 8. BM Acceptance Criteria

- A non-paying player can access all gameplay Relay types.
- A paying player has no direct combat advantage.
- A player never sees purchase UI during boss or defeat countdown.
- Daily missions reward cooperative play, not grind-only behavior.
- Prototype can ship with BM UI hidden and no gameplay dependency breaks.
