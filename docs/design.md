# Relay Defense Design

## Player Fun Target

플레이어가 재미를 느끼는 순간은 세 가지로 잡았다.

1. 패드에 타워를 놓는 즉시 경로 위 적에게 선명한 광선 피드백이 나가는 순간
2. 두 플레이어가 하나의 자원을 공유하며 “이번엔 내가 컨트롤 타워, 다음은 네가 딜 타워”처럼 역할을 나누는 순간
3. 웨이브 사이 짧은 휴식 동안 업그레이드와 상점 선택으로 다음 압박을 준비하는 순간

## Product Direction

최종 앱 목표가 iOS/Android 세로 플레이이므로, 현재 빌드는 PWA 형태의 세로 고정 웹앱으로 만든다. 네이티브 래핑 전에도 모바일 브라우저에서 조작감과 레벨 흐름을 검증할 수 있다.

## Multiplayer

서버는 단일 협동 룸을 권위적으로 시뮬레이션한다. 클라이언트는 빌드, 업그레이드, 구매 명령만 보내고 상태는 서버 snapshot을 받는다. 서버가 없거나 테스트 중이면 클라이언트 로컬 시뮬레이션과 봇 파트너가 즉시 동작한다.

## BM Structure

실제 결제 SDK는 연결하지 않았다. 대신 BM 구조가 붙을 수 있도록 `cores`, 상점 상품, 패스 티어, 일일 미션, 코스메틱 언락 구조를 데이터로 분리했다. 지금 빌드에서는 모두 플레이로 획득 가능한 재화만 사용한다.

## Art Direction

외부 이미지 의존 없이 캔버스에서 일관된 그래픽을 그린다. 어두운 항만/정원 배경, 밝은 빌드 패드, 타워별 고유 색, 적 체력바와 발사선으로 모바일 화면에서도 판독성을 우선한다.
